Understanding Similarity Measures

Similarity Measures are mathematical functions that quantify how alike two entities are based on their characteristics or relationships. These measures are essential in Social Network Analysis (SNA) for identifying relationships, predicting new links, and building recommendation systems.

Key Characteristics

  • Produces numerical values for comparison
  • Indicates degree of resemblance between objects
  • Typically ranges from 0 (no similarity) to 1 (identical)
  • Foundation for recommendation algorithms

Why Do We Need Them?

  • Recommendation Systems: Netflix, Amazon, Spotify use similarity patterns.
  • Link Prediction: Predict future connections like friendships or collaborations.
  • Classification Tasks: Group similar data for predictive modelling.

Common Neighbors

The Common Neighbors measure calculates similarity based on the number of shared connections between two nodes. It reflects how many mutual friends two people have in a network.

Example

Nodes A connects to B and C, while D connects to C.
Neighbors(A) = {B, C}, Neighbors(D) = {C}
Common Neighbors CN(A,D) = |{C}| = 1

Cosine Similarity

Cosine Similarity measures how similar two nodes are based on the angle between their neighbour vectors in network space. It provides a normalized measure of shared connections relative to each node‚Äôs total degree.

  • Perfect Similarity: identical neighbors ‚Üí angle 0¬∞, cosine = 1
  • No Similarity: no shared neighbors ‚Üí 90¬∞, cosine = 0
  • Partial Similarity: acute angle ‚Üí 0 < cosine < 1

Applications

  • Friend recommendation systems
  • Collaboration or citation networks
  • Information retrieval and document similarity

Advantages

  • Degree normalization avoids bias toward high-degree nodes
  • Computationally efficient for large networks
  • Interpretable 0‚Äì1 similarity range

Limitations

  • Considers only local neighborhoods
  • Ignores weighted edges
  • Does not use node attributes

Jaccard Similarity

Jaccard Similarity quantifies how similar two nodes are based on the proportion of shared neighbours to the total unique neighbours. It is normalized and ideal for sparse real-world networks.

Applications

  • Social media friend suggestions
  • Link prediction in evolving networks
  • E-commerce recommendations
  • Biological network analysis

Advantages

  • Handles node degree differences well
  • Produces interpretable normalized values (0‚Äì1)
  • Excellent for sparse networks
  • Easy to explain and implement

Limitations

  • Penalizes high-degree nodes
  • Only captures direct neighbors
  • Ignores node attributes and weights
